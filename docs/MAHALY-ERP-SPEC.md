## 2. SaaS — Super Admin

- دومين: `mahalyerp.com` — سوبر أدمن: `/auth-s` (بدون زر تسجيل في الهبوط)
- كل منشأة: `companyX.mahalyerp.com` + **قاعدة بيانات منفصلة** + credentials خاصة
- اسم مستخدم **فريد عالمياً** (مثل `hany@magyfashon`)
- باقات: فروع + يوزرز + موديولات + موبايل
- دفع: فوري، Paymob، InstaPay، فيزا، محافظ — شريط تحذير + «ادفع الآن»
- تجميد بعد تأخير (مثلاً 10 أيام)، تفعيل يدوي بعد الدفع
- ترحيل DB: سنة/موسم مالي جديد بأرصدة افتتاحية (عملاء، موردين، أصناف) — حد 3 قواعد غير نشطة
- عمولة منصة على دفع الأقساط (حساب محل + حساب شركة الساس على فوري)
- تذاكر + شات دعم

## 3. ERP — موديولات عامة

- **حسابات:** شجرة، قيود، شيكات، بنوك، خزائن، أصول، تقارير
- **مخازن:** أصناف، مقاس/لون، تحويلات، مركبات، صور
- **مشتريات / موردين / عملاء**
- **مبيعات:** نقدي، آجل، تقسيط، POS، حجوزات، عروض أسعار، مرتجعات، طرق دفع متعددة على فاتورة واحدة
- **HR:** هيكل، موظفين، بصمة، شيفتات، سلف، كشف رواتب شهري تلقائي (سجل تاريخي)
- **تصنيع:** BOM، صرف خط إنتاج، مقارنة صرف vs قائمة، استلام تام
- **تقارير** شاملة
- **فواتير ضريبية** + ربط مصلحة الضرائب (ETA)

## 4. قواعد التصميم الأساسية

### فرع vs مخزن
- **الفرع** = منفذ البيع/الشراء/POS الوحيد
- **المخازن** للتخزين والتحويل فقط — لا تظهر في شاشات البيع/الشراء

### الموسم (Season)
- يمر في الشراء، البيع، التقارير، حد الطلب
- حد الطلب يُطبق على **الموسم الحالي المفتوح** فقط

### المستخدمون
- تكويد غير محدود؛ **دخول متزامل** = عدد الاشتراك فقط + رسالة عربية واضحة

### أنواع المنشأة (إعدادات)
- محل فقط / محل+مكتب / محل+مكتب+مصنع

## 6. تخصيصات الملابس (تفصيل التنفيذ)

### مصروفات (3 شاشات)
1. دفعات موردين (كاش/شيك/أوراق دفع)
2. مصروفات عامة (بنود)
3. رواتب + سلف (سلفة/منحة/راتب؛ سلفة تقسيط على أشهر)

### مورد
- مجموعة، قسم (حريمي/رجالي/أطفال/أحذية/شنط)، واتساب، جرد أسبوعي PDF تلقائي
- Custom Fields لكل شاشة

### فاتورة شراء
- مورد + موسم + براند؛ بحث ذكي؛ مصفوفة مقاس×لون
- رفض صنف غير تابع للمورد/الموسم؛ إنشاء صنف من الفاتورة

### صنف + حد طلب
- حد طلب = نسبة من المبيعات من إجمالي الوارد → تنبيه → «مطلوب تكرار» → أمر شراء → واتساب → «أوامر الشراء»

### خصم أوكازيون (مورد)
- خصم على المتبقي → سعر شراء جديد + سعر عرض → POS + كشف مورد + باركود

### عروض عامة
- خصم سعر بيع فقط — لا يؤثر على شراء/كشف مورد

### أوردر (Order)
- مسح باركود + موظف → إيصال → تحميل في بيع/تحويل/مرتجع/جرد

### جرد وتسوية
- فعلي vs دفتر؛ تسوية عجز/زيادة

### POS / بيع
- POS ملء شاشة: بيع / تبديل / تحصيل
- بيع باركود fullscreen
- تقسيط بأنظمة مكودة؛ حجز؛ عرض سعر؛ دليفري
- عدة بائعين؛ تحويل تلقائي عند نقص رصيد

### عملاء
- ضامنين، متابعة متأخرين، SMS/WhatsApp جماعي، إعادة هيكلة أقساط
- تبديل بضاعة (مرتجع + شراء + فرق)

### ورديات / صندوق
- فتح/غلق؛ تسليم لأمين خزنة؛ تقرير يومية كاشير

### تقارير خاصة
- حركة أصناف؛ جرد بضاعة عميل (مكتب/أمانة)； متابعة شيكات
